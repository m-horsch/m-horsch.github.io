title:  "Isn't it a bit lame to use look-up table methods?"
date:   2025-09-11
---
If you enjoy solving puzzles, a look-up table (LUT) Toroidal solver might seem pretty lame.  Lame, because looking up the answer to a puzzle is not as fun as figuring it our yourself.  And if a computer is equipped with a LUT method to solve Toroidal, can we even claim that it is solving the puzzle?  Probably not.  There's no reasoning, logic, or skill involved in looking up the answer.

On the other hand, I do need a very fast way to obtain solutions to Toroidals.  For example, my game site gives a target for the number of moves the player should aim for.  The current batch of puzzles were compiled before I implemented LUT, so it uses an estimate for the optimal number of moves.  But at some point, I will replace the estimate with the exact solution length, at least for 3x3 Toroidals.
